# v0.1 — Zero-key core + voice (PRD / build plan)

**Status:** plan (pre-/evaluate-plan)
**Source specs:** `docs/ARCHITECTURE.md`, `docs/ROADMAP.md` (Phase 0.1), `providers/CONTRACTS.md`,
`skills/manifest/schema/manifest.schema.json`
**Goal:** turn the scaffold into a *verified working tool* for the smallest genuinely-useful
slice — render a real video with no keys, then add ElevenLabs voice + word-level captions.

## In scope (5 work items)

### WI-1 — Golden render (zero-key, keyless)
The Remotion compositor renders the demo manifest to MP4 with no API keys.
- **AC1.1** `npm install` in `remotion/` succeeds on M4 (Node ≥18, chromium headless shell).
- **AC1.2** `echo '{"manifest_path":"examples/manifest.example.json","format":"16:9","out_path":"examples/out/demo-16x9.mp4"}' | node providers/compositor/remotion.mjs` exits 0 and writes a valid MP4.
- **AC1.3** `ffprobe` confirms 1920×1080, duration within ±0.5s of the manifest's summed scene durations, ≥1 video stream.
- **AC1.4** Same for `9:16` → 1080×1920. Both files non-trivial size (>50 KB).
- **AC1.5** Visual spot-check: TitleCard + StatCard scenes render text on brand background (read a extracted frame).
- **AC1.6** `brand.font` actually loads (via `@remotion/google-fonts` or a bundled font) — the extracted frame shows the brand font, not a silent system fallback.

### WI-2 — ElevenLabs TTS adapter
`providers/tts/elevenlabs.mjs` implementing the `tts` contract.
- **AC2.1** stdin `{text, voice, tone?, prev_text?, next_text?, out_path}` → stdout `{audio_path, duration_ms, word_timings[]}`.
- **AC2.2** Key read from env `ELEVENLABS_API_KEY` or `.video-factory/config.json`; **never** from args/JSON.
- **AC2.3** Uses an endpoint that returns character/word timestamps; maps them to `word_timings:[{word,start_ms,end_ms}]`.
- **AC2.4** `duration_ms` measured via ffprobe on the produced audio (not estimated).
- **AC2.5** No key → exit non-zero with a clear stderr message (no stack-only crash).
- **AC2.6** Idempotent: if `out_path` exists and is valid, may return it without re-billing.
- **AC2.7** I/O discipline: **only** the JSON response on stdout; all logs/progress to stderr; exit 0 on success. (A stray stdout write breaks the orchestrator's parse.)
- **AC2.8** `tone` shapes delivery (voice settings / style); `prev_text`/`next_text` passed for natural scene-join prosody when the endpoint supports them.
- **AC2.9** Voice resolution: use the request `voice` if given, else the configured default voice (AC4.5), else a documented hard-coded fallback voice id.

### WI-3 — Word-level animated captions wired end-to-end
The existing `Captions` component is fed real timings.
- **AC3.1** Producer attaches `word_timings` (and `_audio`) to each scene from the TTS step.
- **AC3.2** Compositor stages the audio into `remotion/public/` and the scene plays `<Audio>`.
- **AC3.3** A scene with `captions:"auto"` + timings renders the active word highlighted, synced to audio (frame check at a known timestamp shows the expected active word).

### WI-4 — Minimal `:setup` (ElevenLabs only)
- **AC4.1** Prompts for / accepts `ELEVENLABS_API_KEY`, writes to gitignored `.video-factory/config.json`.
- **AC4.2** Validates via `GET https://api.elevenlabs.io/v1/user`; reports ✅/❌ with reason.
- **AC4.3** Sets `providers.json` `tts.default=elevenlabs` only on successful validation.
- **AC4.4** Re-run merges (doesn't clobber other config); secret never echoed beyond last 4 chars.
- **AC4.5** Records a **default voice id** (prompt or sensible default) into `.video-factory/config.json` so the producer has a voice without per-scene config.

### WI-5 — End-to-end real short
- **AC5.1** From the demo manifest: TTS produced per scene, timings measured, captions wired, rendered to MP4 (16:9 + 9:16).
- **AC5.2** Output has synced VO + animated captions; a neutral viewer sees a coherent ~15s short.
- **AC5.3** `project.json` reflects produced state per scene (resume-safe).
- **AC5.4** **TTS-first write-back**: the producer writes each scene's *measured* `duration_ms` into the manifest before render, so the compositor times scenes to real audio (no estimate drift).
- **AC5.5** Manifest `quality_gate` enforced after TTS: leading/trailing silence ≤ `silence_max_ms` and WPM within `speaking_rate_wpm`; a breach is flagged (regenerate or surface), not silently shipped.

## Out of scope (later phases)
B-roll/clip gen, image gen, music/SFX, avatar, reframe, DaVinci finish, semantic b-roll
retrieval, full robustness of `:video` orchestration. (ROADMAP 0.2+.)

## Acknowledged limitations (intentional in v0.1)
- **Human-gate-before-spend** (ARCHITECTURE Phase B) is a *product* feature; in v0.1 the
  operator running the build IS the gate. The full reviewable-manifest-then-approve UX is 1.0.
- **Zero-key captions**: the `say` adapter returns no word timings, so the keyless path renders
  *without* animated captions. Captions are a voice-tier feature in v0.1 (ElevenLabs timings);
  an `stt` fallback for keyless/timingless audio is deferred to 0.2.

## Dependencies & sequence
WI-1 (keyless, unblocks the render path) → WI-2 (TTS) → WI-3 (captions need WI-2 timings) →
WI-4 (`:setup`, parallel to WI-2/3) → WI-5 (integrates all). WI-4 can proceed in parallel.

## Test strategy (gate — no "looks done")
Each AC verified against a **throwaway fixture** (a tmp manifest / tmp out dir), MP4s checked
with `ffprobe` + frame extraction, not eyeballing the command's exit. WI-5 reviewed
adversarially (does it actually look like a real short, or slop?) before declaring done.

## Risks
- Remotion render on M4 (chromium headless shell download, fonts). → WI-1 first, isolate early.
- ElevenLabs timestamp API shape (which endpoint returns word/char timings; mapping). → spike in WI-2.
- Local audio → Remotion (`staticFile`/public staging vs localhost server). → compositor already stages into public/; verify.
